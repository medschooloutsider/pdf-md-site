Proof Overview

Evidence behind the export claims.

pdf.md was not tuned around toy examples. Its export engine and app workflow were tested against difficult real-world academic documents, with the goal of proving something more important than "a file was written": that the result remains usable, inspectable, and repeatable.

The proof story here is practical: difficult source material, controlled parallel OCR, benchmark discipline, and app-surface QA that reflects the actual release workflow.

Corpus

The testing story is grounded in documents that are hard in practice, not just easy in theory.

Mixed source structure

The golden set includes text-layer PDFs, scan-like pages, embedded raster figures, and documents that move in and out of OCR fallback within the same file.

Quality over mere output

The benchmark logic checks route stability, OCR fallback behavior, normalization impact, and repeatability against stored references.

Commercial-release mindset

The point of the harness is not academic curiosity. It is to reduce omission risk before release and make difficult exports reviewable.

Parallel OCR

Parallelism is a product feature here, but it stays explicit and controlled.

Visible profile choice

pdf.md uses controlled parallel OCR profiles instead of one opaque OCR setting. Eligible page work is scheduled in bounded parallel batches so larger mixed documents can move faster while the chosen runtime profile remains visible to the user.

  • Lite: 5 workers for a lighter runtime posture
  • Balanced: 8 workers as the practical default
  • Max: 10 workers for deliberately heavier OCR throughput

Coverage

The benchmark lane sits inside a broader QA system that also tests the visible app.

Unit coverage

Export invariants, output naming, route assignment, audit behavior, and deterministic OCR-observation paths are validated directly.

Headless export coverage

Repeatable headless runs exercise realistic benchmark profiles and longer export paths without requiring the GUI to be present.

App-surface coverage

The packaged app is also automated through diagnostics, preview, routing, repeated export, cancellation, assignment sheets, and markdown review states.

What a buyer should take from this

pdf.md was tested as a document-production tool, not just a file converter. It can handle ugly source material, expose what happened during export, scale OCR work through explicit profiles, and produce markdown meant to be used after the export is done.
